# Security Policy — charEdge

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it responsibly:

**Email:** [security@charedge.app](mailto:security@charedge.app)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Your suggested fix (if any)

## Response Timeline

| Step | Timeframe |
|------|-----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix deployed | Within 30 days (critical: 72 hours) |

## Scope

**In scope:**
- charedge.app web application
- API endpoints under api.charedge.app
- Client-side data encryption
- Authentication and session management

**Out of scope:**
- Third-party services (Polygon, Binance, etc.)
- Social engineering attacks
- Denial of service attacks
- Issues in dependencies with existing CVE patches

## Safe Harbor

We will not pursue legal action against security researchers who:
- Act in good faith
- Avoid accessing or modifying other users' data
- Report vulnerabilities promptly
- Do not publicly disclose before a fix is deployed

## Recognition

We maintain a hall of fame for verified vulnerability reporters (with your permission).

Thank you for helping keep charEdge secure.
